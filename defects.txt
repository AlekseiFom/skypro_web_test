Файл git.html
1. Ошибка во всех командах, они даны для командной строки и не являются командами Git.
основные команды Git:
1.1 git init - расшифровывается как "initialize repository". что переводится как: "Инициализирует новый Git-репозиторий". Превращает обычную папку в репозиторий 1.2 Git, т.е. создаёт в ней скрытую подпапку .git", в которой Git хранит всю историю изменений, коммиты, ветки и настройки.
1.2 git clone - расшифровывается как "clone repository" что переводится как: "Создаёт локальную копию проекта, который уже существует удалённо." Клон включает в себя все файлы проекта, историю и ветки.
1.3 git commit - расшифровывается как "commit changes" Сохраняет текущее состояние файлов (которые были добавлены через git add) в истории репозитория — создаёт новую точку сохранения (коммит)
1.4 git status  расшифровываются как: "проверить статус"  показывает статус изменений как неотслеживаемые, измененные или подготовленные.
1.5 git branch -переводится как: "Ветка" показаны ветви, над которыми ведутся работы на местном уровне.
1.6 git merge - переводится как: "слияние" объединяет линии разработки. Эта команда обычно используется для объединения изменений, внесённых в две разные ветки.
1.7 git pull - расшифровываются как: "вытянуть" обновляет локальную ветку разработки, используя обновления из удалённой ветки.
1.8 git push - расшифровываются как: "толкать, отправить изменения" обновляет удаленный репозиторий всеми фиксациями, сделанными локально в ветке.
