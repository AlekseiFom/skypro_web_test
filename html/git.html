Git

  • git init - расшифровывается как "initialize repository". что переводится как: "Инициализирует новый Git-репозиторий". Превращает обычную папку в репозиторий Git, т.е. создаёт в ней скрытую подпапку .git", в которой Git хранит всю историю изменений, коммиты, ветки и настройки.
  • git clone - расшифровывается как "clone repository" что переводится как: "Создаёт локальную копию проекта, который уже существует удалённо." Клон включает в себя все файлы проекта, историю и ветки.
  • git commit - расшифровывается как "commit changes" что переводится как: "Сохраняет текущее состояние файлов (которые были добавлены через git add) в истории репозитория — создаёт новую точку сохранения (коммит)"
  • git status расшифровываются как: "проверить статус" показывает статус изменений как неотслеживаемые, измененные или подготовленные.
  • git branch -переводится как: "Ветка" показаны ветви, над которыми ведутся работы на местном уровне.
  • git merge - переводится как: "слияние" объединяет линии разработки. Эта команда обычно используется для объединения изменений, внесённых в две разные ветки.
  • git pull - расшифровываются как: "вытянуть" обновляет локальную ветку разработки, используя обновления из удалённой ветки.
  • git push - расшифровываются как: "толкать, отправить изменения" обновляет удаленный репозиторий всеми фиксациями, сделанными локально в ветке.
